MongoDB Database Setup

üóÑÔ∏è About MongoDB: Paisable uses MongoDB with Mongoose to store all core entities: users, transactions, budgets, receipts, recurring transactions, subscriptions, and usage tracking. Use MongoDB Atlas for a managed cloud deployment or self-host MongoDB locally.

Why MongoDB?

  • Flexible schema: Handles transaction data, budgets, receipts, and user subscriptions without rigid migrations.
  • Great DX: Works seamlessly with Mongoose ODM in Node.js/Express backend.
  • Scalable: Atlas free tier (M0) is perfect for development; scale to larger clusters for production workloads.
  • JSON documents: Records mirror the data shapes consumed by the dashboard and API routes.

Option 2 ¬∑ Local MongoDB

  1. Install MongoDB Community Server (macOS: brew install mongodb-community).
  2. Start the daemon (brew services start mongodb-community or systemd).
  3. Create the paisable database via Compass or mongosh.
MONGO_URI=mongodb://localhost:27017/paisable

  1. Navigate to backend directory: cd backend.
  2. Install dependencies: npm install.
  3. Set MONGO_URI in .env file.
  4. Start dev mode: npm run dev.
  5. Create an account in the app ‚Äî you should see new documents inside the users collection.

Troubleshooting

Common issues:
  • Authentication failed: Double-check the username/password embedded in MONGO_URI.
  • IP not allowed: Update Network Access in Atlas so your server/Render IP can connect.
  • Connection timeout: Verify MongoDB Atlas cluster is running and network access is configured.
  • Timeouts: Attempt to connect with mongosh locally to ensure DNS + firewall rules are correct.

Backups & Security

  • Enable Atlas snapshots or schedule mongodump exports for self-hosted deployments.
  • Rotate database user passwords and avoid embedding secrets directly in source code.
  • Restrict Network Access to trusted IP ranges before going live.
