Stripe Keys & Webhooks

1) API Keys

  1. In Stripe Dashboard → Developers → API keys
  2. Copy Secret key → Set as STRIPE_SECRET_KEY in backend .env
  3. Note: Frontend doesn't need Stripe public key for subscription checkout (handled server-side)

2) Products & Prices (Subscription Plans)

Paisable uses subscription-based pricing. Create subscription products and prices in Stripe:

  1. Stripe Dashboard → Products → Create product
  2. Create products for each plan:
    • Basic Plan - Set up recurring subscription (monthly/yearly)
    • Pro Plan - Set up recurring subscription (monthly/yearly)
    • Enterprise Plan - Set up recurring subscription (monthly/yearly)
  3. For each product, create a Price:
    • Choose "Recurring" billing
    • Set billing period (monthly or yearly)
    • Set the price amount
  4. Copy the Price IDs (they start with price_)
  5. Add Price IDs to backend .env: 
    STRIPE_PRICE_ID_BASIC=price_your_basic_plan_price_id
STRIPE_PRICE_ID_PRO=price_your_pro_plan_price_id
STRIPE_PRICE_ID_ENTERPRISE=price_your_enterprise_plan_price_id
Note: Paisable uses subscription-based payments, not one-time credit purchases. Users subscribe to a plan and get access based on their plan limits.

3) Webhook

  1. Stripe Dashboard → Developers → Webhooks → Add endpoint
  2. Endpoint URL: https://YOUR_BACKEND_DOMAIN/api/stripe/webhook (for Render deployment)
  3. Events to listen for:
    • checkout.session.completed - Subscription checkout completed (activate subscription)
    • customer.subscription.updated - Subscription updated (plan change, renewal)
    • customer.subscription.deleted - Subscription cancelled (downgrade to free)
    • invoice.payment_succeeded - Payment succeeded (renewal)
    • invoice.payment_failed - Payment failed
  4. Copy Signing secret → Set as STRIPE_WEBHOOK_SECRET in backend .env
Important: The webhook endpoint must be accessible from the internet. For local testing, use Stripe CLI to forward webhooks to your local server.
